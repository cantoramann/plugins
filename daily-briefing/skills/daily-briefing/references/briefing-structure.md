# Briefing Structure Specification

The final briefing is a single markdown file with the following sections.

## File Naming

`briefing-YYYY-MM-DD.md` (e.g., `briefing-2026-03-01.md`)

## Header

```markdown
# Daily Briefing — [Weekday], [Month] [Day], [Year]
```

## Section 1: The Rundown

5-6 headline items covering the most important stories of the day across domains.

```markdown
## The Rundown

### [Headline 1]
[One paragraph: what happened + sharp take on why it matters. 2-4 sentences.]

### [Headline 2]
...
```

**Selection criteria:**
- At least 3 different domains represented
- Lead with the single most consequential story
- Each item must pass the Pragmatist Test
- Order by impact, not by domain

## Section 2: Feature Stories

2-3 medium-depth articles. Each is a self-contained piece with its own title.

```markdown
## Feature Stories

### [Title: Descriptive, Not Clickbait]

[300-500 words. Structure: Context → News → Analysis → Implication.]

---

### [Title]

[300-500 words.]
```

**Selection criteria:**
- Stories that deserve more than a headline but less than a deep dive
- Must add genuine analysis beyond what a headline conveys
- Prioritize stories where the "why" is more interesting than the "what"

## Section 3: The Deep Dive

One long-form article exploring a single topic with intellectual depth.

```markdown
## The Deep Dive

### [Title]

[500-800 words. Structure: Hook → Backstory → Current development → Hidden connections → Implications.]
```

**Selection criteria:**
- The Cross-Pollinator selects this topic
- Must teach something genuinely non-obvious
- Backstory or hidden context is essential — this isn't just a longer news article
- The reader should finish with a new mental model, not just new facts

## Section 4: Developing Stories (conditional)

Short updates on stories from previous briefings that have material new developments. Only include this section if the Cross-Pollinator's editorial plan contains developing story updates. Skip entirely on the first briefing or when there are no updates.

```markdown
## Developing Stories

### [Story Headline] — Update
[2-4 sentences. Structure: "When we last covered this [date], [state]. Since then, [what changed]." Direct, no re-explaining the full backstory — the reader already knows it.]

### [Story Headline] — Update
[2-4 sentences.]
```

**Selection criteria:**
- Only stories with material new developments (new official actions, new data, resolution)
- Maximum 3 updates per briefing — if more exist, pick the most consequential
- Each update must add genuinely new information, not restate what was covered before
- Frame as progression: "Since we last covered X..." or "X has evolved since [day]..."

## Section 5: Curiosity Corner

2-3 short pieces about cross-domain discoveries, cultural phenomena, or surprising facts.

```markdown
## Curiosity Corner

### [Short Catchy Title]
[100-200 words. The surface story → the deeper insight → why it matters beyond the obvious.]

### [Short Catchy Title]
[100-200 words.]
```

**Selection criteria:**
- Must pass the Curiosity Test: interesting even if you don't care about the surface topic
- At least 1 item should have nothing to do with tech or business
- Cross-domain connections are gold
- Counterintuitive data is gold
- Cultural milestones that reveal change are gold

## Section 6: One More Thing

A single closing item. Something fun, weird, or thought-provoking.

```markdown
## One More Thing

[2-4 sentences. A memorable closing note — the story someone would mention at dinner.]
```

**Selection criteria:**
- Memorable above all else
- Can be light-hearted, but never trivial
- Bonus points if it connects back to something earlier in the briefing

## Footer

```markdown
---

*Generated by Daily Briefing — a multi-agent curation system. Sources searched broadly across publications worldwide.*
```
